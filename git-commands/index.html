Useful Git Commands

A collection of useful git commands, and how and what they can be used for.
- Akhil Arjun

1. git status

Git status command is used to identify what changes are tracked and which among those are staged. 
git status
  • Untracked files are the ones which are newly added and not tracked for next commit
  • Changes not staged are the ones which are tracked but not added to the list for next commit
  • Staged changes are tracked changes ready to be commited

2. git add

Git add command is used to stage changes. 
git add file_name
Add command can also be used to add multiple files. File names should be seperated by space 
git add file_name_1 file_name_2
If you want to add all the files that were changed use dot operator. 
git add .

3. git reset

Git reset is used to unstage a file from the list of files to be commited 
git reset file_name
Reset command can also be used to reset multiple files. File names should be seperated by space 
git reset file_name_1 file_name_2
If you want to reset all the files that were changed use dot operator. 
git reset .

4. git commit

Git commit is used to commit all the staged changes to remote. 
git commit -m "git commit message here"
Commit command can also be used to add unstaged changes and then commit it by passing an additional flag of 'a'. 
git commit -am "git commit message here"
This above command will stage all untracked changes and commit them in one go.

5. git pull

Git pull is used to pull all the latest changes from remote. it can be called with or without options
We can call this method without any options and it will pull all the changes for the current working branch from remote 
git pull
Or we can pull changes from a specified branch on to the current working branch 
git pull REMOTE_NAME BRANCH_NAME

git pull (contd...)

Git pull runs git fetch internally. Fetch command syncs the local repository with the remote repo.
We can also run fetch command for all branches 
git fetch --all

6. git merge

Git merge is used to merge the latest changes from target branch to the working branch 
git merge target_branch
